# `.maestro/learnings/`

Reusable insights extracted from past sessions, issue threads, and PR conversations. Loaded into every Claude Code session in this repo via `CLAUDE.md`.

## Files

- One `.md` per fine-grained learning. Filename is a slugified short summary.
- `INDEX.md` — regenerated by `tools/build_learnings_index.py`. Do not edit by hand.
- `.last-synthesized-at` — timestamp of the most recent scheduled synthesizer run; used by `.github/workflows/maestro-learn.yml` to compute the next window.

## How learnings get added

- **Scheduled batch.** `.github/workflows/maestro-learn.yml` runs daily and opens a PR titled `learnings: <date>` with anything new.
- **On demand.** Inside a Claude Code session, run `/learn` to capture an insight from the current session. Commits directly to the working branch.

Both paths invoke the same synthesizer prompt (`prompts/synthesizer.md`). The default is to skip — most interactions produce no learning.

## How to remove or correct a learning

These are plain markdown files. Delete one with `git rm`, or open a PR with the deletion. To refine an existing learning rather than delete it, add a new file with `supersedes: [old-file.md]` in its frontmatter — the index regenerator will hide the old file from the main tag groupings and list it under "Superseded".

Then run `python3 tools/build_learnings_index.py` to regenerate the index.
